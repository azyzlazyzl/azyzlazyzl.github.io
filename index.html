配置picgo图床

工具

picgo

下载：Releases · Molunerfinn/PicGo

Typora

下载：Typora 官方中文站

1.Typora配置

打开偏好设置

点击图像，修改为如图所示配置

2.创建新仓库用于存放图片

创建新仓库

新仓库名为Pictures

3.生成tokens

Github头像->Settings->Developer Settings->Personal access tokens->Tokens(classic)->Generate new access Token(classic)

点击Generate token注意这里的token只会出现一次，最好复制后在微信发给自己

4.配置picgo

打开软件，图床设置->Github，填写配置信息，点击确定并设为默认图床


图床名称 默认
仓库名 your-username/blog-images
分支名 main 或 master，看你的仓库设置
Token 填入 GitHub 生成的 Token
存储路径（可选） img/（上传到该目录)
自定义域名 https://cdn.jsdelivr.net/gh/[github用户名]/[仓库名]@main

PS：

坑点1：自定义域名不要用

很慢

接下来安装picgo-plugin-github-plus插件

因为PicGo使用时，只能上传，删除图片时Github仓库无法同步删除，所以要在插件设置安装这个插件。

再进入图床设置->GithubPlus中配置，和上面配github一样

5.链接测试

打开Typora之前配置图像的地方

点击验证图片上传选项

这样是正常的

PS：

坑点2：在上传图片是尽量把加速器关闭，否则会因为证书不对上传失败

这样就完成了配置，使用时将截图复制进md文件，然后去picgo的相册复制图片地址，替换掉原本的地址，这样就可以在Hexo博客上显示图片
