岗位需求

发表于2025-08-04|更新于2025-08-12
|浏览量:

安全服务工程师

初级

  1. 熟悉漏洞扫描、安全加固、日志分析，了解常见攻击手段（如OWASP TOP 10）
  2. 掌握防火墙、IDS、漏洞扫描工具（如AWVS、Nmap）的基础配置
  3. 熟悉Linux/Windows系统操作及安全基线配置

中级

  1. 精通渗透测试、应急响应、内网渗透，能独立完成红蓝对抗项目
  2. 熟悉等保测评、风险评估流程，能制定安全管理制度
  3. 掌握Python/Shell脚本，实现自动化工具或漏洞验证（POC编写）

高级

  1. 精通APT攻击分析、威胁狩猎、溯源反制，具备漏洞挖掘和0day研究能力
  2. 制定企业安全体系架构、数据防护机制，主导重大安全项目（如国家级护网行动）
  3. 熟悉云安全、数据安全合规，具备售前方案设计能力

渗透测试工程师

初级

  1. 熟悉OWASP TOP 10漏洞原理（如SQL注入、XSS），掌握Burp Suite、Nmap、AWVS等工具的基础操作
  2. 了解Linux/Windows系统安全配置，能进行日志分析和安全基线核查
  3. 依赖自动化扫描工具，需协助编写简单渗透报告

中级

  1. 精通内网渗透（域渗透、权限维持）、免杀技术，熟练使用Cobalt Strike、Metasploit等工具
  2. 掌握Python/Java等语言，能编写漏洞利用脚本或优化测试工具
  3. 熟悉云安全（AWS/Azure）、业务逻辑漏洞挖掘，具备APT攻击分析能力

高级

  1. 精通0day漏洞挖掘、APT攻击溯源、红队武器化开发，具备威胁狩猎能力
  2. 熟悉云原生安全、车联网/APP安全攻防，能设计企业级安全防护体系
  3. 制定渗透测试规范，培训团队，输出行业技术标准或学术成果

等保测评

初级

  1. 熟悉Linux/Windows系统安全配置，了解防火墙、IDS等安全设备功能；能操作漏洞扫描工具（如AWVS）
  2. 掌握等保测评基础流程，能依据测评指导书收集证据并整理报告初稿

中级

  1. 精通等保2.0标准，能独立编制测评方案、设计测试用例；熟悉云安全（AWS/Azure）及数据安全合规要求
  2. 掌握网络设备调试（华为/思科）、数据库安全加固、渗透测试工具（Burp Suite）
  3. 必须持有中级等保证书，CISP/CCNP等证书优先

高级

  1. 制定企业安全合规体系，设计数据出境安全评估方案；精通APT防御、威胁狩猎等前沿技术
  2. 熟悉ISO27001、GDPR等国际标准，具备售前方案设计及行业标准制定能力
  3. 必须持有高级等保证书，需满足论文发表、省级科研项目或国家标准参编等学术成果

应急响应

初级

  1. 掌握防火墙、IDS配置，能操作漏洞扫描工具（如AWVS、Nmap）；熟悉日志分析与安全基线核查
  2. 负责7×24小时监控，处理预警事件，编写日报和交接记录

中级

  1. 精通渗透工具（Burp Suite、Metasploit），能独立分析APT攻击链；
  2. 掌握威胁狩猎（Threat Hunting）和MITRE ATT&CK框架，可开发检测规则
  3. 熟悉等保3级标准、SIEM平台（如Splunk、ELK），具备日志和流量分析能力
  4. 必持证书：CISP-PTE、OSCP

高级

  1. 制定企业安全合规体系（如等保三级、数据出境评估）；
  2. 设计零信任架构、云安全防护方案
  3. 精通红队武器化开发、APT溯源反制，具备漏洞挖掘和0day研究能力
  4. 必持证书：CISSP、CISP-PTS

代码审计

至少精通一门或多门编程语言，Java、PHP、Python需求最高，其次为 C/C++、Go、ASP.NET和 JavaScript

熟悉常见开发框架，如 Java 的 SSM/SSH、PHP 的 ThinkPHP/Yii等，并了解其安全特性和漏洞模式

需掌握 Shell、Python 或 Ruby 等脚本语言，用于自动化审计和工具开发

必须熟练使用 Fortify SCA、Checkmarx CxEnterprise、RIPS、Armorize CodeSecure 等主流静态扫描工具。

部分岗位要求掌握 GoASTScanner、Bandit（Python审计）等

能结合工具与人工审计，降低误报率，提升漏洞检出精度

深入理解 OWASP TOP 10漏洞（如 SQL注入、XSS、CSRF、文件上传、反序列化等）的原理、利用手法及修复方案

部分岗位要求熟悉 Windows/Linux 系统机制、内存破坏漏洞（如栈溢出、UAF）及逆向分析能力

云安全（容器、微服务）、移动端（Android/iOS）、区块链智能合约审计经验为加分项

能够编写专业审计报告，清晰描述漏洞原理、风险等级及修复建议

具备SDL（安全开发生命周期） 实践经验，参与制定安全编码规范

在 GitHub、博客、先知社区发布原创工具或技术文章

CTF 获奖、省市级攻防演练攻击队成员经历

发表安全领域论文或参与国家课题研究（如奇安信安全研究员岗位）

总结：

技术栈聚焦：优先掌握 Java/PHP 审计 + Fortify/Checkmarx 工具链，辅以 Web 漏洞原理深度理解。

竞争力构建：通过 CTF 实战、开源项目贡献或漏洞挖掘 积累可验证的成果。

职业发展：向 SDL 流程建设 或 云原生安全 方向拓展，提升架构级能力

SDLC

一、初级岗位（1-3年经验）

  1. 基础技术能力
    • 熟悉 OWASP Top 10 漏洞原理（如SQL注入、XSS、CSRF）及修复方案，具备基础渗透测试能力
    • 掌握至少一门编程语言（Python/PHP/JavaScript），能阅读代码并协助审计
    • 了解自动化工具链（如SAST/DAST）的基本使用，辅助漏洞扫描与报告生成
  2. 流程参与与协作
    • 参与安全测试、代码审查，协助分析测试结果并提供修复建议
    • 熟悉 Docker容器安全 和云环境（如M365）的基础防护
  3. 软技能
    • 良好的英语读写能力（部分岗位需与海外团队协作）

二、中级岗位（3-5年经验）

  1. 核心技术深化
    • 精通 Java/Go/C++ 等至少一门语言，能独立完成源码审计和自动化工具开发（如Fuzz测试框架）
    • 主导 DevSecOps工具链集成（IAST/DAST/SAST/RASP），优化误报率并推动CI/CD嵌入
    • 掌握 威胁建模 和 安全设计评审，识别业务逻辑漏洞（如越权、数据篡改）
  2. 流程建设与落地
    • 制定 安全编码规范（如CERT、OWASP ASVS），推动开发团队执行
    • 负责漏洞全生命周期管理（发现→修复→复测），建立漏洞知识库
  3. 行业经验与资质
    • 金融、互联网等行业项目经验优先，熟悉 ISO 27001/GDPR/等保2.0 等合规标准
    • 持有 CISP/CISSP 认证者更具竞争力

三、高级岗位（5年以上经验）

  1. 架构与治理能力
    • 主导企业级 SDLC/DevSecOps流程建设，制定安全制度并推动跨部门落地
    • 设计 安全解决方案（如数据加密、TEE可信执行环境、零信任架构），覆盖硬件（嵌入式系统/HSM）和云原生场景
  2. 风险治理与创新
    • 体系化梳理业务风险，构建 监控与应急响应机制，主导攻防演练
    • 研究前沿技术（如AI对抗样本、区块链安全），输出防护方案并赋能产品
  3. 团队与影响力
    • 组织 安全培训，提升研发团队安全意识和编码水平
    • 具备 CVE/CNVD漏洞提交 或开源项目贡献经历

四、专家/总监岗（5-10年经验）

  1. 战略级规划
    • 规划企业级 SDL流程与安全体系，整合工具链（SAST/DAST/IAST/RASP）并制定技术路线
    • 设计 长效风险治理机制，覆盖核心链路风险挖掘→事前防御→事后复盘
  2. 跨领域融合
    • 熟悉 硬件安全（SoC/TrustZone/Secure Boot）及物联网协议防护
    • 推动 安全合规认证（如IEC 62443），支撑客户侧安全验收

  3. 行业领导力
    - 要求 金融/头部互联网公司背景，薪资达 60-90K·16薪（上海）

逆向/病毒分析

一、初级岗位（0-2年经验）

  1. 基础技术能力
    • 掌握基础逆向工具（如IDA Pro、OllyDbg、Wireshark）使用，能进行静态/动态分析；
    • 熟悉病毒/木马通信原理和常见加解密算法，了解基础加壳脱壳技术；
    • 至少掌握一门编程语言（Python/C/C++），能编写简单分析脚本。
  2. 漏洞与恶意代码分析
    • 理解OWASP Top 10漏洞原理，能复现基础漏洞并输出报告；
    • 协助完成恶意样本基础分析，提取特征规则。
  3. 软技能
    • 良好的文档撰写能力（分析报告、修复建议）；
    • 英语读写能力（部分岗位需处理国际安全事件）。

二、中级岗位（2-5年经验）

  1. 深度逆向能力
    • 精通X86/ARM汇编指令集，能独立完成复杂样本（APT木马、文档漏洞）逆向分析；
    • 掌握高级调试技术（如内核调试、反混淆），熟练脱壳并还原核心算法；
    • 熟悉Hook框架（Frida/Xposed），实现动态行为监控。
  2. 威胁情报与漏洞研究
    • 主导APT攻击链追踪溯源，提取TTPs（战术、技术、程序）；
    • 分析物联网/车联网协议（BLE/ZigBee）漏洞，输出POC/EXP。
  3. 工具开发与集成
    • 开发自动化分析工具（沙箱、检测规则生成系统）；
    • 优化企业级检测平台（如集成YARA规则）。
  4. 行业资质
    • 持有OSCP/CISP-PTE等认证，或提交过CVE/CNVD漏洞；
    • 金融/物联网行业项目经验优先。

三、高级岗位（5年以上经验）

  1. 架构级分析能力
    • 设计对抗高级威胁的实战化防护体系（如可信执行环境TEE、零信任架构）；
    • 主导智能硬件（嵌入式系统/MCU）固件安全方案，解决通信/OTA安全问题。
  2. APT对抗与情报体系
    • 构建企业级威胁情报平台，整合ATT&CK框架实现攻击链可视化；
    • 组织国家级攻防演练，制定应急响应与取证溯源流程。
  3. 跨领域融合
    • 研究AI安全技术（对抗样本防御、模型窃取检测）；
    • 推动硬件安全方案（Secure Boot、TrustZone）落地。
  4. 团队与影响力
    • 主导安全产品研发（如杀毒引擎、自动化分析平台）；
    • 发表技术论文、开源工具或担任CTF赛事评委。

总结建议

  1. 技术纵深路径：
    • 初级 → 工具链熟练度（IDA+Python） + 漏洞复现能力；
    • 中级 → 深度逆向（内核/协议） + 威胁情报体系构建；
    • 高级 → 架构设计（TEE/零信任） + 跨领域融合（AI/硬件）。
  2. 竞争力构建：
    • 积累 APT分析报告 或 开源工具贡献（如GitHub反病毒引擎项目）；
    • 考取 OSEE/OSCP 认证，参与 国家级攻防演练。
  3. 新兴方向：
    • 车联网安全（ECU逆向、车载协议分析）；
    • AI对抗技术（模型防御、生成式恶意代码检测）。
